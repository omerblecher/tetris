  <what-built>
    Complete Phase 1 Tetris engine:
    - Pure TypeScript engine with SRS rotation, 7-bag randomizer, lock delay, T-spin detection
    - Scoring: line clears (100/300/500/800 × level), T-spins, B2B, combo, perfect clear, drop bonuses
    - Canvas 2D renderer with offscreen pre-rendered neon glow textures (no shadowBlur per frame)
    - useGameEngine hook: engine in useRef, rAF loop with delta time, score/level/lines in React state via events
    - Keyboard controls: arrows, space, Z, C
    - window.game console API for testing
  </what-built>
  <how-to-verify>
    Run `npm run dev` and open http://localhost:5173 in Chrome.

    **Functional verification (play through these scenarios):**

    1. **Basic movement**: Arrow keys move piece left/right without passing through walls. Down arrow soft drops.

    2. **SRS rotation — I-piece near left wall**:
       - Use `game.debug.setNextPiece('I')` in console, let piece spawn
       - Move piece fully to the left wall (5× `game.moveLeft()`)
       - Press Up arrow to rotate — piece should rotate (kick away from wall)
       - If piece cannot rotate near walls, SRS wall kicks are broken

    3. **SRS rotation — I-piece near right wall**:
       - Spawn I-piece, move fully right, rotate — should succeed with kick

    4. **Hard drop + ghost piece**:
       - Space bar should instantly drop piece to the ghost position
       - Ghost piece (dim, same color) should preview exactly where the piece lands

    5. **Hold mechanic**:
       - Press C to hold current piece — held piece type should be... (observable from game state)
       - Press C again on the next piece — should swap
       - Press C twice in a row on same piece — should only hold once (holdUsed flag)

    6. **Line clearing**:
       - Fill a row using console: hard drop pieces strategically, or use `game.debug.setNextPiece`
       - When a row is complete, it should clear and rows above should fall down
       - Score should increase (100 × level for a single)

    7. **Level progression**:
       - Start a game and clear 10 lines — level should increment to 2
       - At level 5+, gravity should be noticeably faster than level 1

    8. **Lock delay**:
       - Let a piece land (don't hard drop) — you should have ~500ms to move/rotate it
       - Moving/rotating near the floor should extend the lock timer (up to 15 times)
       - After 15 moves on the floor without descending, piece should lock immediately

    9. **Game over**:
       - Let the board fill up — game should stop, "GAME OVER" overlay should appear
       - "PLAY AGAIN" button should reset the board and score to 0

    10. **Console API sanity check**:
        ```javascript
        game.state.activePiece    // Should return piece snapshot
        game.state.ghostPiece     // Should return ghost at landing row
        game.state.board          // Should return 20×10 array
        game.score                // Should return current score
        game.level                // Should return current level
        game.debug.printBoard()   // Should print ASCII board to console
        ```

    **Visual verification:**
    - Pieces have distinct neon colors (cyan I, blue J, orange L, yellow O, green S, purple T, red Z)
    - Canvas background is dark (#0d0d1a or similar)
    - Active piece has full-brightness neon glow
    - Ghost piece is same color but translucent (~25% opacity)
    - No visual flickering or dropped frames on an empty board

    **Performance check (optional but recommended):**
    - Open Chrome DevTools → Performance tab
    - Record 10 seconds of gameplay
    - Confirm frame rate stays at 60fps — no dropped frames on an empty board

    **Known Phase 1 limitations (NOT bugs):**
    - No animated line clear (Phase 2)
    - No hold or next-piece panel in the UI (Phase 2 visual polish)
    - No auto-repeat (DAS/ARR) for keyboard — single key press moves one cell (Phase 2)
    - No mobile touch controls (Phase 2)
