Add an animation state machine to `CanvasRenderer`. The render method must accept a `dt` parameter to advance animation timers.

**1. Add types and state at the top of the class (before constructor):**
```typescript
type AnimationType = 'lineClear' | 'lockFlash' | 'levelUp';

interface AnimationState {
  type: AnimationType;
  elapsed: number;
  duration: number;
  rows?: number[];             // lineClear: which rows to flash
  cells?: [number, number][];  // lockFlash: mino [col, row] positions
}
```

Add `private animations: AnimationState[] = [];` as a class field.

**2. Add three public trigger methods:**
```typescript
triggerLineClear(rows: number[]): void {
  this.animations.push({ type: 'lineClear', elapsed: 0, duration: 100, rows });
}

triggerLockFlash(cells: [number, number][]): void {
  this.animations.push({ type: 'lockFlash', elapsed: 0, duration: 50, cells });
}

triggerLevelUp(): void {
  this.animations.push({ type: 'levelUp', elapsed: 0, duration: 300 });
}
```

**3. Update render() signature to accept dt:**
Change `render(state: GameState): void` to `render(state: GameState, dt: number): void`.

Inside render(), at the very beginning (before clearRect), advance and prune animations:
```typescript
this.animations = this.animations.filter(a => {
  a.elapsed += dt;
  return a.elapsed < a.duration;
});
```

**4. Add animation rendering AFTER all game elements (board cells, ghost, active piece), BEFORE game-over overlay:**
```typescript
for (const anim of this.animations) {
  const progress = anim.elapsed / anim.duration; // 0.0 → 1.0
  switch (anim.type) {
    case 'lineClear': this.drawLineClearFlash(anim.rows!, progress); break;
    case 'lockFlash': this.drawLockFlash(anim.cells!, progress); break;
    case 'levelUp':   this.drawLevelUpFlash(progress); break;

```

**5. Add the three private draw methods:**

```typescript
private drawLineClearFlash(rows: number[], progress: number): void {
  const { ctx } = this;
  const alpha = (1 - progress) * 0.85; // fade out from 0.85 to 0
  ctx.fillStyle = `rgba(255, 255, 255, ${alpha})`;
  rows.forEach(r => {
    ctx.fillRect(0, r * CELL_SIZE, COLS * CELL_SIZE, CELL_SIZE);
  });
}

private drawLockFlash(cells: [number, number][], progress: number): void {
  const { ctx } = this;
  const alpha = (1 - progress) * 0.7; // fade out from 0.7 to 0
  ctx.fillStyle = `rgba(255, 255, 255, ${alpha})`;
  cells.forEach(([c, r]) => {
    if (r >= 0 && r < ROWS && c >= 0 && c < COLS) {
      ctx.fillRect(c * CELL_SIZE, r * CELL_SIZE, CELL_SIZE, CELL_SIZE);
    }
  });
}

private drawLevelUpFlash(progress: number): void {
  const { ctx } = this;
  const alpha = Math.sin(progress * Math.PI) * 0.4; // pulse in then out
  ctx.fillStyle = `rgba(255, 200, 255, ${alpha})`;
  ctx.fillRect(0, 0, this.canvas.width, this.canvas.height);
}
```

NOTE: drawGameOverOverlay() is kept as-is for now. It will be replaced by a React DOM overlay in Plan 02-06.

**3. Update onPieceLock handler to trigger lock flash:**
The lock flash needs the mino positions of the piece that just locked. These positions are available from the engine state *before* the engine clears them. However, `onPieceLock` fires inside `lockPiece()` just before `clearLines()` runs — so the board already has the locked piece cells, but `activePiece` is about to be set to null.

The mino positions must be captured from the engine state at the time the event fires. Add a `lockedCells` ref to capture them:

```typescript
// Add ref inside useEffect, before engine.on() calls:
let lastLockedCells: [number, number][] = [];

// Update the existing onPieceLock handler:
engine.on('onPieceLock', (pieceType, _tSpin) => {
  // Capture locked mino positions from engine state BEFORE next piece spawns
  const state = engineRef.current?.state;
  if (state?.activePiece) {
    lastLockedCells = state.activePiece.minos.map(([dc, dr]) => [
      state.activePiece!.col + dc,
      state.activePiece!.row + dr,
    ] as [number, number]);
  }
  rendererRef.current?.triggerLockFlash(lastLockedCells);
  setDisplayState(prev => ({ ...prev, nextPieces: engineRef.current?.state.nextPieces ?? [] }));
});
```

IMPORTANT NOTE about timing: `onPieceLock` fires inside `lockPiece()` before `this.activePiece = null`. So `engine.state.activePiece` will still return the piece snapshot at that moment. Verify this works by checking TetrisEngine.lockPiece() — it fires `onPieceLock` at line ~231 before setting `this.activePiece = null` at line ~248.
