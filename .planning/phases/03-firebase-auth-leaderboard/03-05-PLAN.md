  <how-to-verify>
    Run `npm run dev` and open http://localhost:5173 in your browser.

    **1. Guest experience (start without signing in):**
    - Verify the AuthHeader is visible at the top with a "SIGN IN WITH GOOGLE" button
    - Verify the leaderboard panel is visible in the right side (shows "No scores yet. Be first!" or existing scores)
    - Play a game until game over
    - Verify the game-over overlay shows a "SIGN IN WITH GOOGLE TO SAVE YOUR SCORE" button (prominent, not a link)
    - Verify the LEADERBOARD button on game-over is NOT disabled (cursor changes on hover)
    - Verify PLAY AGAIN restarts the game without requiring sign-in

    **2. Sign-in flow:**
    - Click "SIGN IN WITH GOOGLE" in the header
    - Verify a Google account picker popup appears (not blocked by browser)
    - Sign in with a Google account
    - Verify the header now shows your Google display name and a "SIGN OUT" button
    - Verify the guest sign-in button is gone from the game-over overlay

    **3. Score submission + personal best:**
    - Play a game and reach game over
    - If this is your first game as this user, verify "NEW PERSONAL BEST!" banner appears
    - Verify your score appears in the leaderboard panel (may take a moment for onSnapshot to fire)
    - Check the Firebase Console -> Firestore Database -> scores collection to confirm your document exists with correct uid, displayName, and score

    **4. Real-time leaderboard:**
    - With the game open, open another browser tab to the same localhost URL
    - Sign in with a different Google account (or observe as a guest)
    - Play a game in the original tab and reach a high score
    - Verify the leaderboard in BOTH tabs updates automatically without page refresh

    **5. Auth persistence:**
    - Close the browser tab entirely (not just the tab — close and reopen)
    - Navigate back to http://localhost:5173
    - Verify you are still signed in (display name in header) without clicking sign-in again

    **6. Sign out:**
    - Click "SIGN OUT" in the header
    - Verify header returns to showing the "SIGN IN WITH GOOGLE" button
    - Verify leaderboard still shows (read access remains for guests)

    **7. Firestore index (if query fails):**
    - Check the browser console for any `FAILED_PRECONDITION` error with a Firebase Console URL
    - If present, click the URL to create the missing index — wait ~1 minute for it to build, then refresh

    **Known acceptable states:**
    - If .env values are placeholder/missing, the app may show a Firebase config error in console — fill in real values from Firebase Console
    - If Google sign-in is blocked by a popup blocker, check browser popup permissions for localhost
