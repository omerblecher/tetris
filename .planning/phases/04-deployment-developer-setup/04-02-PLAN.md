  <how-to-verify>
Complete these steps in order. Each step depends on the previous.

**Step 1 — Create GitHub repository (2 min)**
1. Go to https://github.com/new
2. Repository name: `tetris` (or your preferred name)
3. Visibility: **Public**
4. Do NOT initialize with README, .gitignore, or license — the repo must be empty
5. Click "Create repository"
6. Copy the repository URL shown (format: `https://github.com/YOUR-USERNAME/tetris.git`)

**Step 2 — Add remote and push (Claude will do this — provide the URL)**
Tell Claude your GitHub repo URL (e.g., `https://github.com/yourusername/tetris.git`).
Claude will run:
```bash
git remote add origin https://github.com/YOUR-USERNAME/tetris.git
git push -u origin main
```
Verify: GitHub repo page shows all commits and files.

**Step 3 — Deploy to Vercel via dashboard (5 min)**
1. Go to https://vercel.com → click "Add New Project"
2. Click "Import Git Repository" → connect your GitHub account if not already connected
3. Select the `tetris` repo you just created
4. Vercel will auto-detect Vite. Confirm settings:
   - Framework Preset: Vite
   - Build Command: `npm run build`
   - Output Directory: `dist`
5. **BEFORE clicking Deploy** — scroll to "Environment Variables" section and add all 6 vars from your local `.env`:
   - `VITE_FIREBASE_API_KEY`
   - `VITE_FIREBASE_AUTH_DOMAIN`
   - `VITE_FIREBASE_PROJECT_ID`
   - `VITE_FIREBASE_STORAGE_BUCKET`
   - `VITE_FIREBASE_MESSAGING_SENDER_ID`
   - `VITE_FIREBASE_APP_ID`
6. Click "Deploy"
7. Wait for build to complete (usually 1-2 min)
8. **Copy the production URL** — format: `https://tetris-XXXXX.vercel.app`

**Step 4 — Add Vercel URL to Firebase Authorized Domains (CRITICAL — do not skip)**
If you skip this step, Google sign-in will fail with `auth/unauthorized-domain` (popup opens and immediately closes).

1. Go to https://console.firebase.google.com
2. Select your Firebase project
3. Left sidebar → Authentication → Settings tab
4. Scroll to "Authorized domains" → click "Add domain"
5. Enter your Vercel domain WITHOUT https:// (e.g., `tetris-abc123.vercel.app`)
6. Click "Add"

**Step 5 — Verify the live deployment**
1. Open your Vercel URL in a browser
2. Confirm the game loads and renders correctly (neon board visible)
3. Click "Sign in with Google" — confirm the popup opens and completes successfully
4. Play a game and verify score appears on the leaderboard
5. Auto-deploy check: make a trivial change, commit and push → Vercel redeploys automatically

**Step 6 — Update live URL placeholders**
Replace `YOUR-PROJECT.vercel.app` in both files with your actual Vercel URL:
- `CLAUDE.md`: update the `<!-- Live URL: ... -->` comment
- `README.md`: update both occurrences in the header and badge

```bash
git add CLAUDE.md README.md
git commit -m "docs: update live URL placeholder with actual Vercel URL"
git push origin main
```
